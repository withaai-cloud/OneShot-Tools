The 404 error means Vercel can't route to your function. Two issues fixed:

## Changes Made

### 1. Fixed vercel.json Routing
```json
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/api/index"
    }
  ]
}
```

This uses the modern Vercel routing (no legacy `builds` config).

### 2. Fixed api/index.py
- Removed `os.chdir()` that was causing issues
- Simplified to just import and expose the app

### 3. Fixed Template Path Detection
- `app.py` now tries multiple locations for templates
- Works whether called from api/ or root

## Quick Update

**Just replace these 3 files:**
1. `vercel.json`
2. `api/index.py`  
3. `app.py`

Then push to GitHub.
